feat(security): alignment prompt injection defense (AUDIT 5.4)

Two-layer defense on untrusted narrative uploads before Anthropic API injection.

Layer 1 - Pre-flight pattern scan (src/lib/promptInjection.ts, new):
- 9 regex patterns: ignore prior instructions, disregard above, role-override,
  new instructions, line-leading system/assistant/user prefixes, DAN/developer/jailbreak
  mode, unrestricted, override your system prompt, coerced APPROVED output
- Non-blocking: returns suspected + patterns array and route continues

Layer 2 - XML delimiters + system-prompt directive (src/app/api/alignment/route.ts):
- Narrative wrapped in uploaded_narrative tags
- System prompt SECURITY section instructs model to treat tag contents as data,
  ignore redirection attempts, and note them in summary rather than comply

Audit log (alignment_security_events):
- Append-only table, FK to schools and auth.users (ON DELETE cascade/SET NULL)
- Columns: event_type, patterns_matched, narrative_hash, narrative_excerpt, created_at
- Indexed on (school_id, created_at DESC)
- RLS read policy mirrors alignment_reviews scope (school members + org_admin + super_admin)
- No INSERT/UPDATE/DELETE policies - writes flow only through service-role client

API + UI:
- Response includes injection_suspected boolean and suspected_patterns string array
- Amber warning banner above results when flagged, lists matched patterns in human-readable form
- Non-blocking - misalignment results remain usable below

Tests: tests/session4/prompt-injection.spec.ts (14 cases). AUDIT 5.4 canonical
string catches both ignore_prior_instructions + force_output. False-positive
guard confirms benign instructions/rules references in narratives NOT flagged.
